# Azure Resource Manager Documentation Knowledge Base

## Conventions

### Resource Model Naming

Do not suffix user-defined ARM resource model names with `Resource` (use `Employee`, not `EmployeeResource`).
This lets `ResourceNameParameter<Model>` auto-derive correct key/segment names without overrides.

### Rule Doc File Naming

Rule documentation files under `website/.../rules/` must be named to match the registered rule name
exactly (including any `arm-` prefix). The frontmatter `title` should also match the filename. The
fully-qualified rule name (e.g. `@azure-tools/typespec-azure-resource-manager/arm-no-record`)
appears only in the "Full name" code block inside the doc. This ensures that `pnpm regen-docs`
generates correct links in `linter.md`.

### Reference Docs Are Auto-Generated

Files under `website/.../reference/` (decorators.md, data-types.md, interfaces.md, linter.md) are
regenerated by `pnpm regen-docs` from `packages/typespec-azure-resource-manager/`. Never edit these
directly — fix the source `.tsp` doc comment and regenerate.

### `@dev` Comments Must Not Be Removed

The `@dev` tag in `.tsp` doc comments is intentional — it prevents descriptions from becoming
defaults on template instantiations. Fix typos inside `@dev` comments, but never delete or convert
them to regular `/** ... */` comments.

| `...EntityTag`                     | `...EntityTagProperty`                        |
| `...ResourceKind`                  | `...ResourceKindProperty`                     |
| `...ResourcePlan`                  | `...ResourcePlanProperty`                     |
| `...ResourceSku`                   | `...ResourceSkuProperty`                      |
| `...ManagedBy`                     | `...ManagedByProperty`                        |
| `...Encryption`                    | `...EncryptionProperty`                       |

## Envelope Properties

The complete list of envelope property spreads (for `resource-type.md`):

- `...ManagedServiceIdentityProperty` — system + user assigned MSI
- `...ManagedSystemAssignedIdentityProperty` — system-assigned only MSI
- `...EntityTagProperty` — eTag for concurrency
- `...ResourceKindProperty` — resource kind for portal UX
- `...ResourcePlanProperty` — marketplace billing plan
- `...ResourceSkuProperty` — SKU-based service level
- `...ManagedByProperty` — management by other resources
- `...ExtendedLocationProperty` — extended locations (e.g. Azure Arc)

Properties inside the `properties` bag (not envelope):

- `...EncryptionProperty` — customer-managed key encryption
- `...AvailabilityZonesProperty` — availability zones
- `...DefaultProvisioningStateProperty` — standard provisioning state

## Common Pitfalls

### Copy-Paste Errors in Doc Comments

List operations frequently have `@template Resource the resource being patched` copied from patch
operation templates. The correct text should be `the resource being listed`.

### Broken `@doc` Template Interpolation

`@doc("Update a {name PrivateEndpointConnection}", ParentResource)` is wrong — the `{name}` must
have a closing brace: `@doc("Update a {name} PrivateEndpointConnection", ParentResource)`.

### Typos Found in Source

- `decorators.tsp`: "containign" → "containing" in `@armLibraryNamespace`
- `decorators.tsp`: "value fo" → "value of" in `@armProviderNameValue`
- `common-types/private-links.tsp`: Missing `)` in `@template MaxLength` descriptions

## Extension Operations

The `Extension` namespace has these operation templates:

| Template                                 | Description                            |
| ---------------------------------------- | -------------------------------------- |
| `Extension.Read`                         | GET operation                          |
| `Extension.CreateOrReplaceAsync`         | Recommended async PUT                  |
| `Extension.CreateOrUpdateAsync`          | Async PUT (create or update)           |
| `Extension.CreateOrReplaceSync`          | Sync PUT                               |
| `Extension.CustomPatchAsync`             | Async PATCH                            |
| `Extension.CustomPatchSync`              | Sync PATCH                             |
| `Extension.DeleteWithoutOkAsync`         | Async DELETE                           |
| `Extension.DeleteSync`                   | Sync DELETE                            |
| `Extension.ListByTarget`                 | List at target scope                   |
| `Extension.ActionSync`                   | Sync POST action                       |
| `Extension.ActionAsync`                  | Async POST action                      |
| `Extension.ActionNoResponseContentAsync` | Async POST action, no response content |
| `Extension.ActionNoContentSync`          | Sync POST action, no response content  |
| `Extension.CheckExistence`               | HEAD to check existence                |

## Build and Regen Commands

- Build: `pnpm -r --filter "@azure-tools/typespec-azure-resource-manager..." build`
- Regen docs: `pnpm regen-docs` from `packages/typespec-azure-resource-manager/`
- Format: `pnpm format` from repo root
- Requires Node.js >= 22.0.0
